Introduction

Glossary

  • Health Monitoring System: the Monitor Application
  • Remote system: the system to be monitored by the Monitor Application
  • GAE: Google App Engine - A Google service is to host web application
  • Google App: a Google online service that integrates many applications such as: domain controller, emails, docs,....

Infrastructure Overview

The Monitor application is web application, that is deployed on Google App Engine (GAE) and it is integrated to a Google App, so that we can utilize some features of Google User Account Management such as SSO

Functional Overview

CMG Monitor application is an online application to check, and monitor other Web application - Remote systems byscheduling, following information of the other systems will be monitored:

  • Alive status
  • CPU, and Memory usage status
  • Component Services status: database connection ( DAO) , LDAP connection
  • Special Functional services status: web statistic, reporting
The Monitor application will receive information above from remote systems, analysis them and then send notification email to relevant people base on a set of predefined conditions:
  • Is system working or not?
  • Is CPU, or Memory (RAM, JVM) usage being exceed of 90%?
  • Is there any component service connection down?(Database connection, LDAP...)
  • Is there any connection too slow? (base on ping , responded time )

For further detail how to setup remote system, please go to setup remote system

If your account in one of monitor groups, and when a remote system in these monitor groups has any problems, there will be an email sent to you in every 15 minutes (in a schedule), until the problem is resolved

How to setup remote system

The monitor application works by reading an XML file generated by your computer that you are monitoring. The main headings to pass back from your client to the Monitor server are:

  • Database - the data sources that you want to monitor, e.g. MS SQL or Oracle
    • name - e.g. Oracle
    • value - e.g. 2012-10-30 15:58:21.0 which is the last Ping time
    • error - any error text, e.g. "no connection" or "server timed out"
    • ping - e.g. "Ping time = 0 milliseconds"- the ping time in milliseconds
  • JVM - health of the JVM and how much memoty is being used from that allocated for the virtual memory
    • Free_Memory - e.g. "452331744" -number of bytes available
    • Total_Memory- e.g. "531103744" -number total of bytes allocated
    • Max_Memory- e.g. "1061814272" - maximum amount of memory available
    • Used_Memory- e.g. "78772000" - total used memory
  • fielsystem - the data sources that you want to monitor, e.g. LDAP or Oracle
    • file_name - e.g. the drive letter of the file location to monitor, e.g. C:\
    • size - e.g. "17G" in total avaialable
    • used - e.g. "12G" used
    • available - e.g. "4.8G" remaining disk space
    • Percentage_used - e.g. 72%
    • mount - e.g. what is this drive labelled as when mounted
  • cpu
    • usage - e.g. "54%"
    • vendor - e.g. "Intel"
    • model - e.g. "Xeon"
    • total - e.g. how many CPU's available
  • cpu_physical
    • cpu_type - e.g. "Mem" or "Swap" to indicate what CPU type is being analysed
    • total - e.g. "2531920" - total available
    • used
    • free
